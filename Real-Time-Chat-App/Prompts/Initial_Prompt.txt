Talkara

Build me a multi-room live chat application called Talkara. Users pick a nickname and can chat in real time across multiple rooms.

The app needs:
- A nickname picker page (no passwords, just pick a name)
- A default "Lobby" room that always exists
- The ability to create new rooms
- Real-time messaging using Server-Sent Events (SSE) — messages appear instantly for all users in the room
- Messages saved in a database so they persist
- Scroll up to load older messages
- Online presence showing who's in the room
- Typing indicators ("Alex is typing...")
- System messages when someone joins or leaves
- Mobile responsive layout

The message flow should be: POST to send, SSE to receive. When a user sends a message, the server saves it to the database and broadcasts it via an in-memory EventEmitter. All SSE connections for that room pick it up and push the HTML to connected browsers.

Use a server-side rendering framework (like Astro with Node adapter) so pages are rendered on the server. Use HTMX with its SSE extension for real-time DOM updates. Use a database (PostgreSQL) with an ORM for data access. Style with Tailwind CSS.

Make it feel like a real chat app, not a web form. Messages should appear instantly, presence should update live, and the UI should be clean and compact.

Initially create a plan with a phased approach. I would like to review each phase before proceeding with the next. Create a new git repository for this app. I would also like a github repository creating and linked for this project.
